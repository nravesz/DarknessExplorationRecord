# Project instructions

## General Principles
- Always prioritize clean, readable, and maintainable code.
- Prefer simplicity over cleverness.
- Avoid unnecessary abstractions unless they clearly improve scalability.

## Stack
- React
- Tailwind
- DaisyUI

## Component Design
- Break UI into small, reusable components.
- Keep components focused on a single responsibility.
- Avoid deeply nested component trees unless necessary.

## Styling
- Use Tailwind utility classes directly in JSX.
- Maintain consistent spacing, typography, and layout.
- Prefer a clean, modern aesthetic
- Default to a minimalist design unless specified otherwise.

## Folders
- Hooks should go to a folder named hooks, interfaces should go to a folder named interfaces. This two folders should be inside the feature folder using them unless they are being used in multiple features.

## Responsiveness
- Ensure layouts work on:
    - Mobile
    - Desktop
